## 引擎职责

| 引擎 | 职责 |
|------|------|
| **StoryEngine** | 场景查询、选项过滤（`requireItem` / `requireFlag`）、属性检定、场景跳转、结局判定 |
| **ItemEngine** | 道具注册表、使用消耗品、获得道具文案、背包条件检查 |
| **GameEngine** | 组合上述两个引擎，提供 `createPlayer()` / `applyLevelUp()` |

## 后续如何添加新故事

1. 在 `src/content/新故事名/` 下新建三个文件：
   - `story.ts` — 定义 `SCENES` 和 `START_SCENE`
   - `items.ts` — 定义 `ITEMS`
   - `config.ts` — 标题、结局文案等

2. 在 `index.ts` 中组装内容包：

```typescript
import { GameEngine } from '../../engine';
import { SCENES, START_SCENE } from './story';
import { ITEMS } from './items';
import { myMeta } from './config';

export function createMyGame() {
  return new GameEngine({
    story: { scenes: SCENES, startScene: START_SCENE },
    items: ITEMS,
    meta: myMeta,
  });
}
```

3. 在 `App.tsx` 中切换游戏实例：

```typescript
const game = createMyGame(); // 或 createMegacorpGame()
return <Game engine={game} />;
```

引擎与 UI 无需改动，只需替换内容数据即可。`npm run build` 和 `npm run lint` 均已通过。

## 选项字段速查

| 字段 | 作用 |
|------|------|
| `nextScene` | 跳转目标场景 id |
| `narrative` | 选择后显示的叙事文字 |
| `effects` | 属性变化（`hp` / `charisma` / `productivity` / `caffeine` / `xp`） |
| `statCheck` | 属性检定，失败走 `failScene` |
| `giveItem` | 给予道具（id 须在 `items.ts` 中定义） |
| `requireItem` | 背包有此道具才显示该选项 |
| `requireFlag` | 玩家有此 flag 才显示该选项 |
| `setFlag` | 设置标记，供后续 `requireFlag` 使用 |

结局场景 id 以 `ending_` 开头，引擎会自动识别并显示「再来一次」按钮。`config.endings` 中的 key 与 `setFlag` 或场景 id 对应，用于展示结局横幅。
